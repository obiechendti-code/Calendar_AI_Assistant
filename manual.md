# GAS行事曆每日秘書：完全安裝手冊

**歡迎使用您的個人 AI 秘書！**
這套工具基於 **Google Apps Script (GAS)** 技術構建，深度串接 **Google Calendar** 自動讀取您的每日行程，並透過 **Google Gemini** 強大的 AI 模型進行智慧摘要、重點整理與優先級排序，最終將結構化的匯報寄到您的信箱。您不需要懂程式碼，只要依照本手冊的步驟「複製、貼上、點擊」，約 **10 分鐘** 即可完成設定。

## 🎒 階段一：準備工作 (Getting Ready)

在開始之前，請先準備好以下兩樣東西（就像做菜前先備料）：

### 1. Google Gemini API Key (AI 的大腦金鑰)

這是免費的，請放心申請。

1. 前往 [**Google AI Studio**](https://aistudio.google.com/)。
2. 登入您的 Google 帳號。
3. 點擊左上角的 **"Get API key"** 按鈕。
4. 點擊 **"Create API key"**。
5. 系統會產生一串以 `AIza` 開頭的亂碼，點擊 **"Copy"**。
    - 📝 **請先暫時貼在您的記事本上，等一下會用到。**

### 2. 確認您的日曆 ID

- 對於大多數人來說，日曆 ID 就是您的 **Gmail 信箱地址** (例如 `yourname@company.com`)。
- 除非您要抓取的是「共用日曆」或「特定專案日曆」，才需要另外查詢 ID。

## 🛠 階段二：建立專案 (Create Project)

1. 前往 [**Google Apps Script 首頁**](https://script.google.com/)。
2. 點擊左上角的 **「+ 新專案」** 大按鈕。
3. 進入編輯器後，點擊最上方的文字 **「未命名的專案」**。
4. 將專案名稱改為：`Gemini AI 每日秘書`，按下「重新命名」。

## 🏗 階段三：部署代碼 (Copy & Paste)

現在我們要建立 **7 個檔案**。請依照下表，一個一個建立。

**操作教學：**

1. 在左側「檔案」列表，點擊 **「+」** 號 -> 選擇 **「指令碼」**。
2. 輸入 **檔名** (參照下表，不要打錯字)。
3. 按下 Enter。
4. **刪除**編輯區內原本的 `function myFunction() {...}` (清空內容)。
5. 將同事提供給您的對應程式碼 **完整複製貼上**。

**檔案清單 (請依序建立)：**

| 順序 | 檔名 (請複製粗體字) | 說明 |
| --- | --- | --- |
| 1 | **Config** | 全域設定檔 (注意：系統會自動補上 .gs，您只要打 Config) |
| 2 | **Prompt** | AI 的指令與人設 |
| 3 | **Workflow_DailyBrief** | 每日寄信的主程式 |
| 4 | **Utils_CalendarParser** | 讀取行事曆的工具 |
| 5 | **Utils_TaskClassifier** | 專案分類工具 |
| 6 | **Utils_Setup** | **(重要)** 設定 ID 與密碼的工具 |
| 7 | **Utils_Debug** | 除錯與測試工具 |

> 💾 小提醒： 全部貼完後，請按一下上方工具列的 磁片圖示 (儲存)，或是按鍵盤 Ctrl + S。
> 

## 🔐 階段四：輸入機密資訊 (Secrets Setup)

為了保護您的隱私（不讓密碼直接寫在程式碼裡），我們使用「一次性設定」的方式。

1. 在左側點擊開啟 **`Utils_Setup.gs`** 檔案。
2. 找到程式碼中的 `const SECRETS = { ... }`區塊。
3. 將您的資訊填入 **引號 `""` 之中**：
    - `CALENDAR_ID`: 填入您的 Email。
    - `GEMINI_API_KEY`: 填入剛剛申請的 `AIza...` 金鑰。
    - `COMPANY_EMAIL`: (選填) 若想寄備份到公司信箱可填，不想填請留空。
    - *(其他欄位若無使用 Notion 可保持空白)*
    
    > 注意： 請保留雙引號，例如 "CALENDAR_ID": "obie@gmail.com", 是正確的。
    > 
4. **執行寫入：**
    - 確保上方工具列的函式選單顯示 **`runFirstTimeSetup`**。
    - 點擊 **「執行」** 按鈕。
5. **確認成功：**
    - 觀察下方的「執行紀錄」。
    - 若出現 **「✅ 環境變數設定完成！」**，代表密碼已存入後台保險箱。
6. **🧹 安全清除 (重要)：**
    - 現在，請把 `Utils_Setup.gs` 裡您剛剛填寫的真實 Key 與 ID **刪除** (回復成空字串 `""` 或原本的提示文字)。
    - **再次存檔**。
    - *這樣一來，即使別人看到您的螢幕或程式碼，也看不到您的密碼了。*

## 🚦 階段五：授權啟動 (Authorization)

這是 Google 的安全機制，第一次執行時會跳出警告，請不用擔心。

1. 在左側開啟 **`Workflow_DailyBrief.gs`**。
2. 上方函式選單改選為 **`sendDailyBriefing`**。
3. 點擊 **「執行」**。
4. **授權流程 (請按圖索驥)：**
    - 視窗跳出「需要授權」 -> 點擊 **[審查權限]**。
    - 選擇您的 Google 帳號。
    - 🚨 **出現紅色警告視窗「Google 尚未驗證這個應用程式」**：
        - 別慌張！這是因為程式是您自己寫的，Google 還不認識它。
        - 點擊左下角的灰色小字 **「進階 (Advanced)」**。
        - 點擊最下方的連結 **「前往 Gemini AI 每日秘書 (不安全)」**。
    - 點擊 **[允許]**。
5. **驗收成果：**
    - 觀察下方執行紀錄。
    - 如果看到 `[Workflow] Email 已交付給：xxxx@gmail.com`。
    - **恭喜！** 請去您的信箱收信，您應該已經收到第一份 AI 匯報了。

## ⏰ 階段六：設定自動排程 (Automation)

最後一步，我們要設定鬧鐘，讓它每天早上自動運作。

1. 將滑鼠移到左側選單，點擊 **「觸發條件」** (圖示是一個鬧鐘)。
2. 點擊右下角的藍色按鈕 **「+ 新增觸發條件」**。
3. 在跳出的視窗中，依照以下設定調整：
    - **選擇要執行的功能**：選 `sendDailyBriefing`
    - **選取活動來源**：選 `時間驅動 (Time-driven)`
    - **選取時間型觸發條件類型**：選 `日計時器 (Day timer)`
    - **選取時段**：選 `上午 07:00 至 08:00` (或您喜歡的時間)
4. 點擊 **[儲存]**。

## 🎉 完成了！

從明天開始，無論您是否有打開電腦，AI 秘書都會在早上 7 點準時將整理好的工作重點寄給您。

### ❓ 常見問題 QA

**Q: 如果我想暫停寄信怎麼辦？**
A: 回到「觸發條件」頁面，把剛剛建立的觸發器刪除即可。

**Q: 為什麼我沒收到信？**
A:

1. 檢查垃圾信件夾。
2. 確認您今天（或測試當天）的日曆上是否有行程？若無行程，系統預設可能不會寄信（視設定而定）。
3. 使用 `Utils_Debug.gs` 進行測試。

**Q: 我想修改 AI 的語氣，例如更活潑一點？**
A: 沒問題！打開 **`Prompt.gs`** 檔案，直接修改裡面的中文敘述，AI 下次執行時就會依照新的人設說話了。
